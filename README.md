## Задача 4 Seq.map

### Текст задачи

Получить список из длин строк, содержащихся в исходном списке

### Алгоритм решения

1.	Запросить у пользователя количество строк с помощью рекурсивной функции readInt, которая проверяет, что введено целое положительное число, и повторяет запрос при ошибке.
2.	Создать ленивую последовательность строк readStrings с использованием seq. При каждом обращении к элементу последовательности выводится приглашение и считывается строка с клавиатуры.
3.	Применить к полученной последовательности функцию Seq.map, передавая ей лямбда-выражение fun s -> String.length s, которое вычисляет длину каждой переданной строки.
4.	Для вывода результата последовательность преобразуется в список с помощью Seq.toList, чтобы содержимое (полученные числа) отобразилось в консоли. Сама обработка данных при этом остаётся в рамках ленивых последовательностей.

# Задание 2

## Задача 4 Seq.fold

### Текст задачи

Найти сумму тех элементов списка, в которых встречается заданная цифра.

### Алгоритм решения

1.	Запросить у пользователя количество элементов списка с помощью рекурсивной функции readInt, которая проверяет, что введено целое положительное число, и повторяет запрос при ошибке.
2.	Запросить искомую цифру с помощью рекурсивной функции readDigit, которая проверяет, что введена ровно одна цифра (от 0 до 9), и повторяет запрос при ошибке.
3.	Создать ленивую последовательность чисел readNumbers с использованием seq и вспомогательной функции ввода readElement. При каждом обращении к элементу последовательности выводится приглашение и считывается целое число с клавиатуры.
4.	Применить к полученной последовательности функцию Seq.fold, передавая ей начальное значение аккумулятора 0 и лямбда-выражение fun acc num -> .... Внутри выражения проверяется, содержит ли строковое представление текущего числа (num.ToString()) заданную цифру. Если содержит, число прибавляется к аккумулятору, иначе аккумулятор передается на следующий шаг без изменений.
5.	Вывести итоговый результат. Поскольку функция Seq.fold сворачивает всю последовательность в одно финальное значение, она сама автоматически инициирует поочередный ввод всех элементов ленивой последовательности. Полученная сумма напрямую выводится в консоль с помощью printfn.

# Задание 3

## Задача 4 

### Текст задачи

Вывести самое длинное название файла в указанном каталоге.

### Алгоритм решения

1.	Запросить у пользователя путь к каталогу с помощью рекурсивной функции readDirectory, которая проверяет существование указанной папки с помощью системного метода Directory.Exists и повторяет запрос при ошибке ввода.
2.	Получить массив полных путей ко всем файлам в этом каталоге с помощью метода Directory.GetFiles. Операция выполняется внутри блока try ... with для безопасной обработки возможных системных ошибок (например, отсутствия прав доступа к системной папке).
3.	Проверить полученный массив на пустоту. Если массив пуст (файлов в каталоге нет), вывести пользователю соответствующее сообщение.
4.	Применить к массиву путей функцию Seq.map, передавая ей лямбда-выражение fun f -> Path.GetFileName(f). Это преобразование отсекает полные пути к файлу, оставляя в последовательности только сами имена файлов с их расширениями.
5.	Применить к полученной последовательности имен функцию Seq.maxBy, передавая ей лямбда-выражение fun name -> String.length name. Данная функция самостоятельно переберет все элементы и вернет то имя файла, длина которого является максимальной.
6.	Вывести найденное самое длинное название файла и его вычисленную длину в консоль.
